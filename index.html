Fractal Brushes

Draw luminous symmetry with living brushes.

Fractal Brushes turns pointer movement into mirrored tendrils, elemental filaments, blended color trails, and export-ready artwork. It runs as a static GitHub Pages site with no backend.

A polished canvas app, not a static sketch.

The drawing engine stays at the center, now framed around expressive brush families, floating menus, precise color selection, brush expansion, a zoom slider, and export presets for finished artwork.

Expanded brush set

Fairy, flame, thunder, silk ribbon, comet trail, nebula smoke, crystal filament, water, ember, frost, and plasma each shape motion differently, with expansion control for spread and density.

Draggable brush menu

Keep the brush picker open beside the canvas, move it where it fits, and switch profile behavior without interrupting the drawing flow.

Circular hue ring

Choose primary and blend colors from a ring picker with separate handles, toggle blending, or jump quickly with preset chips for repeatable palettes.

Floating color panel

The color menu opens under the main controls by default, drags independently, and stays available while the brush menu remains open.

Bottom-right zoom

Use the bottom-right slider to set zoom precisely and read the current zoom percentage from a compact control surface anchored away from the drawing tools.

Artwork exports

Save Current Resolution, 4K 16:9, or 4K Square PNG files with timestamped Fractal Brushes filenames.

Designed for fast experimentation.

Open the app, draw, adjust floating panels, zoom the canvas, and export without accounts or installation.

Main controls stay compact

Logo, undo, redo, export, clear, mirror, brush menu, color menu, and randomize stay grouped in the top-left toolbar.

Panels can stay open

Brush and color menus close independently, so you can tune a palette while comparing luminous and elemental brush behavior.

Static deployment

The landing page and app are plain HTML, CSS, and JavaScript, so GitHub Pages can deploy the repository directly.

Ready for GitHub Pages.

The repository includes a Pages workflow. Push to main, enable Pages with GitHub Actions as the source, and the landing page will publish at the root while the drawing app remains available at /app/.
